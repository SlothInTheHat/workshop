## 3. CONTRIBUTOR VIEW

**Route:** `/workshops/[id]/contributor`

**Access Control:**
- Requires session cookie
- Redirects to `/join?return=` if not logged in
- Creates participant record automatically if doesn't exist
- Auto-assigns role as 'contributor'
- Redirects to `/workshops` if workshop not in 'pre' status

| Feature | Status | Details |
|---------|--------|---------|

**The 5 Sections:**
1. **Goals & Objectives** - What outcomes to achieve

## 4. BACKEND

### 4.1 AI Integration

**Kickoff Summary Generation:**
- Endpoint: `POST /api/ai/kickoff-summary`
- Model: Claude Sonnet 4.5
- Input: title, focusArea, objective, dataSensitivity
- Output: 2-3 paragraph kickoff summary
- Used in: `/workshops/new` (Step 3)

**AI Context Generation:**
- Endpoint: `POST /api/workshops/[id]/context`
- Model: Claude Sonnet 4.5
- Input: Workshop details + all contributor inputs
- Output: Comprehensive context document with:
  - 3-5 common themes
  - Key pain points
  - 3-4 opportunity areas
  - Specific focus areas
  - Constraints and success criteria
- Saved to: `workshops.aiContext`

### 4.2 Database

**Uses Drizzle ORM** with PostgreSQL

**Connection:** `src/lib/db/index.ts`
- Checks `DATABASE_URL` environment variable
- Falls back to in-memory if not set

**Key Schema Files:**
- `preWorkshops` - Main workshop table
- `preParticipants` - Participants
- `contributorInputs` - 5-section inputs
- `artifacts` - Reference materials
- `activityLogs` - Audit trail

**Note:** There are TWO separate workshop schemas:
1. **Pre-workshop schema** (`preWorkshops`, `preParticipants`, etc.) - Used for pre-workshop phase
2. **Live workshop schema** (`workshops`, `participants`, etc.) - Used for live and post-workshop phases

## 5. ACCESS CONTROL

### 5.1 Session Management

**Cookie:** `pw_session`
- HttpOnly, SameSite=lax, 7-day expiry
- Contains: `{ name, role, tenantId }`

**Roles:**
- `facilitator` - Can create workshops, manage participants, view all inputs
- `contributor` - Can only submit their own input

### 5.2 Access Codes

**Environment Variable:** `PHASE1_ACCESS_CODES`
- Format: `FACILITATOR=FAC123,CONTRIBUTOR=CON123`
- Defaults: FAC123 (facilitator), CON123 (contributor)

**Login Flow:**
1. User visits `/join`
2. Enters name and access code
3. Server validates code
4. Sets session cookie with appropriate role
5. Redirects to original destination

### 5.3 Route Guards

| Route | Required Role | Redirect If No Session | Redirect If Wrong Role |
|-------|---------------|------------------------|------------------------|
| `/workshops` | Any | `/join` | - |
| `/workshops/new` | Facilitator | `/join` | `/workshops` |
| `/workshops/[id]/pre` | Facilitator | `/join` | `/workshops/[id]/contributor` |
| `/workshops/[id]/contributor` | Any | `/join` | - (auto-creates participant) |

## 7. ARCHITECTURE SUMMARY

### 7.1 Data Flow

```
1. FACILITATOR creates workshop
   /workshops/new → POST /api/workshops

2. FACILITATOR adds participants
   /workshops/[id]/pre → POST /api/workshops/[id]/participants

3. CONTRIBUTOR logs in
   /join → Sets session cookie

4. CONTRIBUTOR submits input
   /workshops/[id]/contributor → POST/PATCH /api/workshops/[id]/inputs/[uid]

5. FACILITATOR generates AI context
   /workshops/[id]/pre → POST /api/workshops/[id]/context

6. FACILITATOR launches workshop
   /workshops/[id]/pre → PATCH /api/workshops/[id] (status: 'live')
```

### 7.2 Status Lifecycle

```
draft → pre → live → completed

- draft: Initial creation (not used currently)
- pre: Pre-workshop phase, contributors can submit inputs
- live: Workshop is live, inputs are locked
- completed: Workshop finished
```

### 7.3 What Works End-to-End

✅ **Fully Functional:**
1. Facilitator creates workshop
2. Facilitator adds participants
3. Contributors log in with access code
4. Contributors fill and submit 5-section form
5. Facilitator views progress in real-time
6. Facilitator generates AI context from all inputs
7. Facilitator edits AI context
8. Facilitator launches workshop
